### 移行系タスクで AI が旧ファイルを削除できず往復が発生

- **日付**: 2026-05-29
- **Wave**: cc-spec-alignment Wave 3（commands → skills 移行）
- **症状**: `.claude/commands/*.md`（11本）を `.claude/skills/<name>/SKILL.md` へ移行する際、
  新ファイルは Write で作成できたが、旧ファイルを残すと同名スキルが二重登録され競合する。
  しかし憲法（security-commands.md）で `rm`/`mv` は AI 禁止のため削除できず、
  着手後に削除方式を決める往復（手動依頼）が発生した。
- **原因**: 「新規作成」と「旧削除」がワンセットの移行タスクなのに、削除を AI ができない制約を
  着手前に織り込んでいなかった。移行手順の設計に削除ステップの実行主体が欠けていた。
- **回避策**: 移行・リネーム系タスクは **着手時に削除方式を先決め** する。選択肢は
  (a) `git rm` を ask 承認で都度実行、(b) skill のみ作成し旧ファイルは手動削除、
  (c) `/ship` 内で一括 `git rm`。判定表・タスク定義の完了条件に「旧ファイル除去（方式: X）」を明記する。
  add+delete が不可分な単位は、その対を1コミットにまとめると競合ウィンドウを最小化できる。
- **参照回数**: 0
