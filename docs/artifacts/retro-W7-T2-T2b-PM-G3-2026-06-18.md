## 6. ハイライト・教訓

**統括の直接 Bash 操作が「実機投入の本質として代替不可」と確定**。W7-T2b では gd_guard /
initialize_state / accumulate_subagent_tokens / distill-lessons / set_status / build_cost_summary の
6 種類の Bash 呼び出しが必須となった。これは MAGI 合議で「実機投入時の例外」として事前確定済であり、
evidence §6 で明示記録されている。「3 層体制 = 統括 Edit/Write/Bash=0」は **平常時の原則** であり、
実機投入は例外として運用する方が現実的、という運用ルールが確立した。

**立ち止まりトリガーが 3 回機能した**。本セッションで「PM 級判断発生（grader 起動回数不確実性）」
「合議結論を覆す新事実（global_bound 矛盾）」「数値 typo Critical（wc -l vs Read 行数）」のいずれも
事前定義したトリガーで検出 → ユーザー判断仰ぎ → 修正後再進行。トリガーを **作業開始時に宣言** したことで、
自分自身（Sonnet/Haiku 含む）も従う行動規範として機能した。事前宣言の効果が定量的に確認された。

**MAGI 結論を要約段階で歪めるリスクを発見**。W7-T2b Atom B3「実機 1 回 + pytest 代替の組み合わせ」を
選択肢 (I)/(II)/(III) に整理する段で「(II) pytest 代替」と書いてしまい、MAGI 原文と矛盾。
ユーザー指摘で発覚し (III) に修正。**MAGI 原文と要約の差分チェックを Try に追加**（Try #1）。
合議結論の保全は「結論を出すこと」と同等に重要、という教訓が得られた。
