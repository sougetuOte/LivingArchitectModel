## Issue 一覧

### Warning

- **[W-1] [SE]** `_hook_utils.py:149` — `except (OSError, RuntimeError)` の `RuntimeError` 捕捉の意図が不明確。
  現行 CPython の `Path.resolve(strict=False)` は循環 symlink を `OSError`（POSIX `ELOOP` / Windows `WinError`）に変換し、`RuntimeError` は送出しない。`RuntimeError` を残すこと自体はフェイルセーフ（予期せぬ内部エラーでも out_of_root に倒す）として正当だが、コメントが意図を伝えていないため「実際には起きない例外を握りつぶす Dead Code 隣接」に見える。
  → **修正方針**: 例外型は狭めず（hook クラッシュ回避のフェイルセーフを維持）、コメントで `OSError`/`RuntimeError` 各々の捕捉意図を明示する。

### Info（Green State 非阻害・参考）

- **[I-1] [SE]** POSIX 絶対パス（`/etc/passwd`）は Windows で `resolve()` するとカレントドライブ基点（`C:\etc\passwd`）に解決される。W-15 由来ではない既存事象。**W-16（path traversal）で整理**。（両エージェント言及）
- **[I-2] [PG]** docstring の W-15 節に `resolve(strict=False)` で未作成 Write パスも親まで解決される旨を補記すると契約が明確。
- **[I-3] [SE]** out_of_root フォールバック表示のコメントが docstring と inline に分散（情報三箇所）。inline を「see docstring W-15」に集約する将来候補。
- **[I-4] [PG]** `test_hook_utils.py:174` の symlink テスト skip 条件 `except (OSError, NotImplementedError)` の `NotImplementedError` は `Path.symlink_to` では発生しない（dead code）。`except OSError` で十分。
- **[I-5] [SE]** root 内 symlink を将来 `.claude/rules/` 等に構成追加する場合、resolve 後の実体パスが PM/SE パターンにマッチするか要検証（現構成では該当なし・実害なし）。タスク留意事項候補。
