## Atom A1: 「4 者目」の価値軸の確定

**[MELCHIOR]**:
- 4 人目のペルソナ (実装者・ユーザー代弁者等) の追加はプロンプト 1 段落で済み、導入コスト最小
- 視点の幅が形式上広がり、「4 者合議」という語感の座りも良い

**[BALTHASAR]**:
- Reflection 変更率 0% と HGA §3 が示す通り、**同一モデル・同一文脈のペルソナを何人足しても盲点は相関したまま**。難所ほど偽の合意に合流する
- ペルソナ増は出力トークンとフロー重量だけ増やし、実効安全性を増やさない
- 最大のリスクは「4 人いる」という誤った安心感 (ADR-0007 案 4 却下理由「機能している印象を与える」と同型)

**[CASPAR]**: 結論: **4 者目の価値軸は「人数」ではなく「独立性」(文脈独立 + モデル非相関)**。
同一文脈ペルソナの追加は却下。4 者目は独立文脈の検証者でなければ意味を持たない。

## Atom A2: ADR-0007 (gabriel) との関係

**[MELCHIOR]**:
- 「独立文脈 + 異モデル (Sonnet) の検証者」は、まさに ADR-0007 が決定済みの gabriel そのもの
- spec Green + tasks Green 達成想定まで進んでおり、4 者化の実装は「ADR-0007 を完遂する」ことでほぼ達成される。新規 ADR 不要で速い

**[BALTHASAR]**:
- ADR-0007 は **Proposed のまま Phase 6 PM 承認待ち**。4 者化議論が ADR-0007 の再審議に化けると既決の蒸し返し (Zero-Regression 違反)
- gabriel のトリガーは「AoT 適用時のみ」。4 者化の名目で常時起動に広げると ADR-0007 案 3 の却下理由 (過剰投資) を再来させる

**[CASPAR]**: 結論: **4 者化は ADR-0007 の上書きではなく「完成 + 体制上の位置づけの明文化」**。
gabriel のトリガー条件・失敗時挙動 (D-3) は一切変更しない。今回新たに足すのは概念整理と HGA 接続の明文化のみ。
前提条件として **ADR-0007 の PM 承認 (Phase 6) が先行必須**。

## Atom A3: 呼称・概念整理

**[MELCHIOR]**:
- 「第 4 の MAGI = GABRIEL」と正式命名すれば対外的にわかりやすく、ユーザーの「4 者化」の語感にも合う

**[BALTHASAR]**:
- 危険。gabriel を「合議メンバー」と呼ぶと、**合議に混ぜる (同一文脈化する) 誤運用を誘発**する。gabriel の価値は「合議の外にいること」そのもの (ADR-0007 案 2 却下理由 = 文脈染み込み)
- 命名の意匠が既に設計を表している: MELCHIOR/BALTHASAR/CASPAR = 東方三博士 (合議する賢者)、gabriel = 大天使 (外から告げる者)。HGA (守護天使 = 上位オラクル) とも「天使系 = 外から来る」で一貫する

**[CASPAR]**: 結論: **合議は 3 者のまま。体制としては「MAGI 3+1 体制」(3 賢者 + 1 天使) と呼ぶ**。
gabriel は「第 4 の投票者」ではなく「投票しない第 4 の構成員 = 外部検証者」と明文化する。
「4 者化すべきか」への回答は「**合議の 4 者化は否、体制の 4 者化は既に ADR-0007 が実現しており、それを明文化する**」。

## Atom A4: エスカレーション階梯の明文化

**[MELCHIOR]**:
- 「3 ペルソナ (Opus 同一文脈) → gabriel (Sonnet 独立文脈) → Fable (HGA 召喚) → 人間 (PM)」の 4 段階梯を 1 表で明文化すれば、判断の重さと資源配分が一目で対応し、hga-summoning.md の召喚ゲート「MAGI 敵対テストで塞げない破綻」とも接続する

**[BALTHASAR]**:
- 文書重複リスク。階梯の構成要素は 06_DECISION_MAKING (SSOT)・decision-making.md・hga-summoning.md・ADR-0007 D-3 に分散して既に半分書かれている。新規文書は fat 削減 (B-5 骨子 ④) に逆行
- ADR-0007 の BUILDING (SKILL.md / 06_DECISION_MAKING 改修) が未実施。先に階梯だけ書くと実装と文書が乖離する

**[CASPAR]**: 結論: **新規文書は作らない**。ADR-0007 BUILDING 時に既定されている文書更新
(06_DECISION_MAKING.md / decision-making.md / SKILL.md) に**エスカレーション階梯 1 表を相乗り**させる。
本合議の決定は ADR-0007 への小追補 (改訂履歴 + 注記) として記録する。

**[MELCHIOR]**:

**[BALTHASAR]**:

**[CASPAR]**: 結論: v2 の内容は

『新しいセッション/文脈で gabriel 相当プロンプトを手動実行』という軽量代替**」とする。
pipeline の verify には触れない。**v2 更新は本家 gabriel BUILDING 完了後** (動く実装を見てから移植する)。

## AoT Synthesis

**統合結論**: **「合議の 4 者化」は却下、「体制の 4 者化 (3+1) の明文化」を採用**。

1. 合議ペルソナは 3 者のまま変更しない (MELCHIOR / BALTHASAR / CASPAR)
2. gabriel (ADR-0007) を「投票しない第 4 の構成員 = 独立文脈・異モデルの外部検証者」として体制上正式に位置づける — 呼称「**MAGI 3+1 体制**」
3. エスカレーション階梯を ADR-0007 BUILDING の文書更新に 1 表相乗りで明文化する:

| 段 | 主体 | 文脈 / モデル | 役割 |
|---|------|--------------|------|
| 1 | MELCHIOR / BALTHASAR / CASPAR | 同一文脈 / L1 モデル | 発散・議論・収束 |
| 2 | gabriel | 独立文脈 / Sonnet | 敵対検証 (AoT 適用時のみ / ADR-0007 D-3) |
| 3 | Fable 召喚 | 独立セッション / Fable | 塞げない破綻・無条件召喚ゾーン (ADR-0009 / hga-summoning.md) |
| 4 | 人間 (PM) | — | 最終エスカレーション (再 MAGI 上限超過 / abort) |

4. 同一文脈への 4 人目ペルソナ追加・gabriel の常時起動化・トリガー条件変更は行わない
5. etm-diary v2 への反映は本家 gabriel BUILDING 完了後
