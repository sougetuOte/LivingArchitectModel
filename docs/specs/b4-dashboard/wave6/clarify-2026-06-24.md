- 実装コスト +30〜50% を許容

## 4. R2 結果（技術選定）

### Q4: ブランドカラー

**選択**: B. Radix Colors パレットの採用

- Vercel/Radix UI 製の WCAG 適合パレット（ライト・ダーク両対応スケール設計済）
- 12-step scale（1=背景 / 12=高コントラスト文字）
- npm 配布だが LAM の NFR-5（外部依存なし）+ PM 級制約（package.json 変更禁止）に従い、必要な色だけ手動で `<style>` に転記する方針
- 必要セット見込み: gray + blue + green + amber × 各 12 step × 2 セット = 約 96 値 / 4〜6 KB 増（NFR-1 への影響極小）
- 公式: https://www.radix-ui.com/colors

### Q5: CSS フレームワーク

**選択**: D = A 採用（フルスクラッチ pure CSS / L1 推奨）

- 学習コスト 0 / 依存 0 / NFR-1 余裕あり
- CSS カスタムプロパティ + Grid + flexbox で LAM 規模（4 ビュー / 380 行テーブル）に対応
- 複雑化したら Wave 7+ で Pico CSS / Tailwind ビルド済への移行可能

### Q6: ソート/フィルタ JS

**選択**: A. Vanilla JS を inline（L1 推奨 / 約 100〜150 行）

- 依存 0 / NFR-5 と整合
- 列ヘッダクリック → `Array.from(tbody.rows).sort()` → DOM 再挿入
- フィルタ: input → `row.style.display` 切替
- HTMLTableElement.rows: https://developer.mozilla.org/ja/docs/Web/API/HTMLTableElement/rows

## 6. L1 デフォルト固定事項（質問せずに採用）

ユーザー知見不足を考慮し、Lighthouse Accessibility 95+ 達成に必須の項目は L1 推奨でデフォルト採用とした:

| 項目 | デフォルト |
|------|-----------|
| フォント | システムフォント（`system-ui, -apple-system, "Segoe UI", "Hiragino Sans", "Noto Sans JP", sans-serif`） |
| 等幅フォント | `ui-monospace, "SF Mono", Consolas, monospace`（コード/数値表示） |
| コントラスト | WCAG 2.1 AA 準拠（4.5:1 以上 / Radix Colors の step 11-12 を文字に） |
| フォーカス可視 | `:focus-visible` で 2px outline + offset / コントラスト 3:1 以上 |
| セマンティック HTML | `<main>`, `<nav>`, `<section>`, `<article>` の活用 / Wave 5 で指摘された `landmark-one-main` 解消 |
| ARIA | `aria-label`（ソートボタン等）/ `aria-sort`（テーブルヘッダ）/ `aria-live`（フィルタ件数）必要箇所 |
| キーボード操作 | テーブルソートボタン Tab 移動可能 / Enter/Space で動作 |

これらは design.md で具体的に明文化する。
