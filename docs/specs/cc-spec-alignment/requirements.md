# 要件定義: Claude Code 仕様すり合わせ更新（cc-spec-alignment）

最終更新: 2026-05-27 / フェーズ: PLANNING（requirements）/ ステータス: **draft（承認待ち）**

## 1. Problem Statement

最終コミット 2026-03-31 以降、Claude Code のプラットフォーム仕様（hooks / skills / sub-agents /
MCP / settings / permissions）が拡張・一部非推奨化された。差分マッピング
（[00-diff-mapping-summary.md](00-diff-mapping-summary.md)）により、**破壊的な乖離はゼロ**だが、
ドキュメント記述の陳腐化（7件中4件）・新機能の未活用（64件）・要確認事項（5件）が判明した。
本更新は LAM を最新仕様と整合させ、安全に活用可能な新機能を取り込むための計画である。

## 2. Goals / Non-Goals

### Goals
- 陳腐化したドキュメント/コメント記述を事実に合わせる
- 公式仕様との不一致疑い（H-20 等）を裏取りし、必要なら是正する
- LAM の Memory Policy について公式 `memory:` 機構との関係を明確化する
- 採用価値のある新機能を選定し、後続フェーズの対象を確定する

### Non-Goals（非スコープ）
- 64件の新機能を網羅的に実装すること（選定したもののみ対象）
- hooks の Python 移行アーキテクチャ自体の再設計（v4.6 系で完了済み、本件は整合のみ）
- 要件書・ADR の API 書式依存箇所以外の改訂（Upstream First 原則: 要件・ADR は更新不要）
- Claude Code 本体のバージョン固定やダウングレード

## 4. 採用候補の新機能（FR-5 の暫定提案 — design で確定）

ユーザー方針「公式と乖離しない／低中リスクは着手可」に沿った保守的な暫定優先度。

| 優先 | 機能 | 領域 | 理由 |
|:----:|------|------|------|
| 高 | `paths`（パターン自動起動） | Skills | LAM の権限等級・対象ファイル運用と親和的 |
| 高 | `permissionDecision: "defer"` | Permissions | 既存 ask ガードを settings 委譲でき柔軟化 |
| 中 | `model` 上書き（Skills/Sub-agents） | 両 | 本セッションで実証した「下位モデル委譲」を恒久化 |
| 中 | `when_to_use` / `allowed-tools` | Skills | 起動精度とツール最小権限の明示 |
| 低 | `isolation: worktree` / `background` | Sub-agents | 並列レビューの隔離。要検証 |
| 低 | `effort.level` 入力 / 新 hook ハンドラ | Hooks | 効果は大きいが影響範囲も広く慎重評価が必要 |

## 6. 設計判断: Memory Policy（MAGI System 適用）

### 判断内容
LAM の Subagent Persistent Memory（`.claude/agent-memory/<name>/`、CLAUDE.md 指示で自発書込）を、
公式 `memory: project` 機構（同一パスを正式サポート）へ移行するか、現行の自前運用を維持するか。

**[MELCHIOR（推進）]**: 公式 `memory:` へ寄せるべき。同一パスを公式サポートしており、フロントマター
1行（`memory: project`）で宣言できる。CLAUDE.md の長い運用指示に依存せず、Claude Code 本体の
ライフサイクル管理に乗れる。Upstream First 原則とも合致し、将来の機能拡張も享受できる。

**[BALTHASAR（批判）]**: 移行にはリスクがある。①公式 `memory:` の書込タイミング・スコープ
（user/project/local）の挙動が LAM の想定（レビュー知見の蓄積）と一致する保証がまだ裏取りされていない。
②同一パスでも公式機構が既存ファイルをどう扱うか（上書き/追記）不明。③ C-2/C-3 は影響度 低〜中で、
現行運用は破綻していない。拙速な移行は Zero-Regression Policy に反する。

**[CASPAR（調停）]**: 結論 = **「段階的移行を前提に、まず裏取り（FR-3）を完了させ、その結果を条件に
`memory: project` を採用する」**。即時全面移行も現状維持固定もせず、(a) 公式 `memory:` の挙動を
公式ドキュメントで確認 →(b) 自前運用の想定と一致するなら 8エージェントに `memory: project` を付与し
CLAUDE.md の記述を「公式機構を利用」へ更新 →(c) 一致しない/不明なら現行運用を維持しつつ CLAUDE.md の
陳腐化記述（「公式機能ではない」）のみ事実に合わせて修正。いずれの分岐でも FR-1.2 は満たされる。

### Reflection
致命的な見落とし: なし。ただし本決定は PM級であり、**最終確定には人間承認が必須**（休憩明けに確認）。
暫定方針として CASPAR の段階的移行案を design の前提に置く。
